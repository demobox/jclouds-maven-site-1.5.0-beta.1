40  /**
41   * VirtualSystemSettingData defines the virtual aspects of a virtual system through a set of
42   * virtualization specific properties.
43   *
44   * VirtualSystemSettingData is also used as the top level class of virtual system configurations.
45   * Virtual system configurations model configuration information about virtual systems and their
46   * components. A virtual system configuration consists of one top-level instance of class
47   * VirtualSystemSettingData that aggregates a number of instances of class
48   * {@link ResourceAllocationSettingData}, using association {@link ConcreteComponent).
49   * <p>
50   * Virtual system configurations may for example be used to reflect configurations of:
51   * <ul>
52   * <li>virtual systems that are defined at a virtualization platform
53   * <li>virtual systems that are currently active
54   * <li>input requests to create new virtual systems
55   * <li>input requests to modify existing virtual systems
56   * <li>snapshots of virtual systems
57   * </ul>
58   * 
59   * @author Adrian Cole
60   * @author grkvlt@apache.org
61   * @see <a href="http://schemas.dmtf.org/wbem/wscim/1/cim-schema/2.22.0/CIM_VirtualSystemSettingData.xsd">CIM_VirtualSystemSettingData</a>
